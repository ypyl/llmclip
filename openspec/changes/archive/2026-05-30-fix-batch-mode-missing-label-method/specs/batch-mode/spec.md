## ADDED Requirements

### Requirement: Batch item labels generated via ContextPresentationService
`SendBatchToLLMCommand` SHALL use `ContextPresentationService.GetLabelFromContextItem()` to generate labels for batch context items, rather than attempting to call a nonexistent method on `ContextManager`.

#### Scenario: Label generation for file context item
- **WHEN** a batch item is a file path (e.g., `C:\docs\report.txt`)
- **THEN** the label is generated by `ContextPresentationService.GetLabelFromContextItem()` using the file icon and path formatting
- **THEN** the label is attached to each response message via the `batchContextItem` property

#### Scenario: Label generation for URL context item
- **WHEN** a batch item is a URL (e.g., `https://example.com`)
- **THEN** the label is generated by `ContextPresentationService.GetLabelFromContextItem()` with the web icon
- **THEN** the label is attached to each response message via the `batchContextItem` property

#### Scenario: Label generation for plain text context item
- **WHEN** a batch item is plain text (truncated to 50 characters with ellipsis)
- **THEN** the label is generated by `ContextPresentationService.GetLabelFromContextItem()` with the text icon
- **THEN** the label is attached to each response message via the `batchContextItem` property

#### Scenario: Service dependency injection
- **WHEN** `SendBatchToLLMCommand` is instantiated in `App.ahk`
- **THEN** it receives a `ContextPresentationService` instance via constructor injection
- **THEN** the `cps` variable (already instantiated in `App.ahk`) is passed as the new parameter
